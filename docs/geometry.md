# 立方体の幾何

Sudocube で「新しく考える必要があるのはここだけ」という核心部分。どの面のどのマスが、どの面のどのマスと辺・頂点で繋がるか（= 同じ数字になるか）を定義する。

## 基本方針：辺の対応表は手で書かない

「前面の右端は右面の左端に上下反転で対応して…」を12辺ぶん手で書くと必ずバグる。代わりに**3D座標の一致で機械的に対応を生成する**。

1. 立方体を 3D 空間 `[0,8]^3` に置く
2. 6面それぞれのマス `(row, col)` に3D座標（マスの位置）を割り当てる
3. **3D座標が一致する（誤差 < 1e-6）マス同士を同一視する**

こうすると、辺の向きの反転も、頂点で3面が集まることも、幾何が自動的に解いてくれる。人間は「立方体を正しく組む」ことだけ考えればよい。

## 6面の座標系（検証済み）

各面の原点・row方向ベクトル・col方向ベクトルを、立方体の各面（外向き法線）に忠実に定義する：

```js
const GEO = {
  U: { origin: [0, 0, 8], rowDir: [0, 1, 0], colDir: [1, 0, 0] }, // z = 8（上面）
  D: { origin: [0, 0, 0], rowDir: [0, 1, 0], colDir: [1, 0, 0] }, // z = 0（下面）
  F: { origin: [0, 0, 0], rowDir: [0, 0, 1], colDir: [1, 0, 0] }, // y = 0（前面）
  B: { origin: [0, 8, 0], rowDir: [0, 0, 1], colDir: [1, 0, 0] }, // y = 8（後面）
  L: { origin: [0, 0, 0], rowDir: [0, 0, 1], colDir: [0, 1, 0] }, // x = 0（左面）
  R: { origin: [8, 0, 0], rowDir: [0, 0, 1], colDir: [0, 1, 0] }, // x = 8（右面）
};

// マス (r, c) の3D座標:  point = origin + r*rowDir + c*colDir   (r, c = 0..8)
```

境界のマス（index が 0 または 8）が立方体の辺・頂点にちょうど乗るため、3D座標の一致を取るだけで辺・頂点の共有が生成される。

## twins と peers

この幾何から、パズルのロジックが使う2つの純粋関数を導く：

- **`twins(face, r, c)`**：そのマスにくっついている双子マス（立方体の同じ辺／頂点にある別の面のマス）を返す。0〜2個。
  - 面の内側 → 0個
  - 辺の上 → 1個（2面が共有）
  - 頂点 → 2個（3面が共有）
- **`peers(face, r, c)`**：そのマスと同じ数字になってはいけない相手すべて。= 自分の面の行・列・ブロック + 双子それぞれの面の行・列・ブロック。

ソルバはこの `peers()` だけを見ればよく、アルゴリズム本体は通常の数独と変わらない（[generation.md](generation.md) 参照）。

## 独立変数は386個（検証済み）

辺・頂点で共有されるマスを同一視して畳むと、独立に決めるべき変数の数は次のようになる：

| 種類 | マス数 | 変数数 |
|---|---|---|
| 面内部（辺に接しない 7×7 × 6面） | 294 | 294 |
| 辺（角を除く・2面共有） | 168 | 84 |
| 頂点（3面共有） | 24 | 8 |
| **合計** | **486** | **386** |

実験でこの内訳がちょうど再現されることを assert 済み。

## 頂点の具体例

角 `xyz = [0,0,8]` に3面が集まり、完成盤面では3つとも同じ値になる（検証時の実例では値 8）：

```
U 面 (r=0, c=0) = 8
F 面 (r=8, c=0) = 8   ← この3マスが双子（同じ頂点）
L 面 (r=8, c=0) = 8
```
