# 進捗と次のステップ

## 完了

- **実現可能性の検証**：seedなしDFSで完成盤面が存在することを確認（386変数・8ミリ秒・416分岐）。制約による「解ゼロ」ではないと確定
- **幾何の確定**：6面の座標系と、3D座標一致による辺・頂点の自動対応（[geometry.md](geometry.md)）
- **データ構造の確定**：6面の数字 + `twins()` / `peers()`（[data-structure.md](data-structure.md)）
- **プロジェクト scaffold**：Vite 8 + React 19 + TypeScript 6

## 次のステップ

### 1. 生成ロジックの本実装（最優先）

`src/core/` に以下を実装する：

- `board.ts` — `Board` 型・面ごと文字列との相互変換・`setCell`（双子同期込み）
- `geometry.ts` — GEO定義から `twins()` / `peers()` を構築（3D座標一致ベース）
- `solver.ts` — DFS + MRV・`solve()` / `countSolutions(board, cap)`
- `generator.ts` — 完成盤面生成 → 掘る → ユニーク解の問題

テスト（テストランナーは未導入・vitest を想定）：

- geometry：ユニーク変数 386・内部294・辺84・頂点8・各頂点3セル
- solver：完成盤面が生成でき、全面の行/列/箱が {1..9}、全双子ペアが同値
- generator：生成した問題が `countSolutions == 1`

**この段階で「掘ってユニーク解の問題が作れるか」「難易度が成立するか」を事実で確認する**（[generation.md](generation.md) の未検証項目）。

### 2. UI（3D表示 + 正面スナップ回転）

- three.js / react-three-fiber で立方体を描画
- ドラッグで自由回転 → 手を離すと最寄りの面がカメラ正面にスナップ（quaternion の slerp / tween）
- 6面それぞれの「正面姿勢」を定義し、ドラッグ終了時に最寄りへ吸着
- 正面に来た面で数字を入力（WebGL 上に直接ではなく、正面の面に HTML グリッドを重ねる方針）

### 3. 仕上げ

- 面の向き（見た目）の調整：「正面の上端 = 上面の下端」のような、プレイヤーの直感に合う向きへの合わせ込み（存在性には影響しない純粋な表示の問題）
- 難易度設定・盤面の保存/読み込み・演出

## 未決の論点
